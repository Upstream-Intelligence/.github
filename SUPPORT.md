# Support

How to get help with Upstream Intelligence projects.

## I'm a customer of the Upstream Care Intelligence Platform

For product issues, billing, account questions, or anything affecting your production deployment:

- **Help:** [upstream.cx/support](https://upstream.cx/support)
- **Email:** hello@upstream.cx
- **Security or PHI concerns:** security@upstream.cx (see [SECURITY.md](SECURITY.md))

Do not file GitHub issues for production product questions. The public repos here are for open-source tools, integrations, and methodology, not the product itself.

## I hit a bug in an open-source repo

File a GitHub issue in the affected repo. Use the bug report template. Include:

- Repo name and commit hash
- What you ran
- What you expected
- What happened
- Environment (OS, runtime version)

Repos:

- [upstream-mcp](https://github.com/Upstream-Intelligence/upstream-mcp/issues) — MCP server bugs
- [upstream-skills](https://github.com/Upstream-Intelligence/upstream-skills/issues) — Claude Code skills bugs
- [upstream-community](https://github.com/Upstream-Intelligence/upstream-community/issues) — open ML methodology bugs
- [awesome-payer-risk](https://github.com/Upstream-Intelligence/awesome-payer-risk/issues) — broken links or wrong entries

## I have an idea or question

Use GitHub Discussions on the relevant repo. Discussions exist for:

- General Q&A
- Show-and-tell (operators sharing how they use the tools)
- Ideas (feature suggestions before they're full issues)

If you're unsure which repo, post in [Upstream-Intelligence/.github discussions](https://github.com/orgs/Upstream-Intelligence/discussions) once enabled, or in `upstream-community`.

## I'm not sure where to file

Email hello@upstream.cx and we'll route it.

## Response times

- Customer support: same business day on weekdays
- Bug reports on public repos: triaged within 3 business days
- Discussions: best effort
- Security: see [SECURITY.md](SECURITY.md)
